# 博客开发需求说明（交给 Gemini）

## 1. 项目目标

- 构建一个**静态优先（Static-First）**的个人博客。
- 目标风格：**极简、白底、企业级高级质感**，非花哨模板风。
- 目标体验：首屏有强记忆点（高质量空间动效），整体仍保持加载轻、运行快。
- 目标工程属性：可长期运营、可维护、可扩展、可快速迁移部署。

## 2. 部署与迁移要求

- 当前部署目标：Cloudflare（Pages/静态托管优先）。
- 后续迁移目标：个人 VPS。
- 迁移方式必须简单：
  - 代码托管私有 GitHub。
  - VPS 侧可通过 `git pull` + build/deploy 完成迁移。
- 禁止强耦合某家平台私有能力（避免锁定）。

## 3. 域名与路由

- 主域名：`blog2.dwill.top`
- 文章 URL 形态：`/posts/<slug>/`（如 `blog2.dwill.top/posts/xxx/`）
- 标签页：`/tags/` 与 `/tags/<tag>/`
- 关于页：`/about/`

## 4. 内容生产方式

- 内容源以 **Markdown** 为主（不是直接让 AI 生成纯 HTML 作为源）。
- 支持工作流：
  - 我给文本 -> AI 生成/润色 Markdown -> 放入内容目录 -> 构建发布。
- Markdown 需支持：
  - 表格
  - 数学公式（LaTeX/KaTeX）
  - 代码高亮
  - ECharts（按需加载，不拖慢首屏）
  - 图片插入
- 暂不考虑视频能力（可预留扩展点，但不做重功能）。

## 5. SEO 与可检索性

- 必须具备：
  - `sitemap.xml`
  - RSS
  - 规范化 meta（title/description/canonical/open graph）
  - 结构化数据（Article / Breadcrumb / WebSite）
  - 合理 robots 策略
- 要求文章能被搜索引擎稳定抓取和索引。

## 6. 性能要求（硬指标倾向）

- 静态输出，减少运行时 JS。
- 首屏核心内容优先可见（动效不阻塞可读内容）。
- 动效必须有降级策略：
  - `prefers-reduced-motion` 生效。
  - 低性能设备可简化渲染密度。
- 目标方向：
  - JS 体积小
  - 首屏渲染快
  - 动效帧率稳定

## 7. UI/UX 设计总原则

- 基调：白底、克制、精致、企业高级质感。
- 禁止问题：
  - 过度左倾排版
  - 字号/按钮/卡片比例失衡
  - 暗色重口风格（默认应是浅色高质感）
- 要求：
  - 桌面与移动端都要协调
  - 排版层次清晰，留白有节奏
  - 交互反馈细腻但不喧宾夺主

## 8. 首页首屏动效（关键需求）

- 不是普通 banner，要有“震撼但高级”的空间感。
- 动效序列要求：
  1. 开场先出现清晰可读的一句话：`OVER THE RAINBOW`
  2. 这句话由同一字形**原地解析**成粒子（不是前景字消失、背后再冒一层字）
  3. 粒子向 360° 扩散（不是只往左上/右上）
  4. 粒子收敛到近乎消失
  5. 再次爆散
  6. 爆散的同时中心从“远处空心圆”逐步拉起成立体漏网/漏斗结构（强 3D 空间感）
  7. 最终停留在终态：漏网与粒子持续运动，但**不重播剧情，运动的范围要大，并且具有强烈立体感**
- 粒子视觉要求：
  - 彩色
  - 小粒子占多数，中粒子少，大粒子极少
  - 有前后景深，不是平面撒点

## 9. 动效工程化要求

- 动效应组件化，可替换（未来可切换不同 Hero Effect）。
- 推荐结构：
  - `HeroEffectHost`（调度）
  - `effects/<effect-name>`（具体实现）
  - 配置文件切换激活效果
- 后续若换动效，新旧可共存，不要互相破坏。

## 10. 图片与静态资源策略

- 图片存储可先走仓库内静态资源（小规模可行）。
- 需预留后续迁移到对象存储/CDN的路径。
- 构建时做基础优化（压缩、尺寸合理化、懒加载策略）。

## 11. 代码质量与可维护性

- Windows 环境开发，所有文本文件必须 UTF-8（避免中文乱码）。
- 代码结构清晰、命名可读、避免魔法常量散落。
- 新需求应可增量扩展，不推翻现有架构。

## 12. 交付物要求

- 完整可运行项目。
- 一份部署说明（Cloudflare + VPS 两套流程）。
- 一份内容写作规范（Markdown 模板、frontmatter 规范、slug 规则）。
- 一份“动效参数说明”（方便后续自己调速度/密度/相机感）。

## 13. 验收标准（Done Definition）

- 视觉：首页达到“简约 + 高级 + 有记忆点”的观感。
- 技术：构建通过，核心页面可访问，路由与 SEO 文件齐全。
- 运营：写一篇新文章可在 5 分钟内完成“写作 -> 入库 -> 构建 -> 部署”。
- 迁移：项目可在新机器上通过 `git pull` 快速恢复并发布。
